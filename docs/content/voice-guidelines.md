# Voice & Writing Guidelines — Resources & Guides

## Purpose

Supporting content exists to rank for question-intent queries and help users complete tasks on ConvertMyStuff tools. It is not generic blog filler.

## Differentiation test

Every article must:

1. Link prominently to a **primary tool** the reader can use in one click.
2. Include **worked examples with real numbers** that match formulas in `src/lib/tools/logic/`.
3. Cover **practical edge cases** in examples or common-mistakes / FAQ blocks.
4. Highlight **client-side / privacy** benefits where true (PDF, text, developer tools).
5. Avoid paragraphs that repeat the title or pad word count without teaching.

## Tone

- Clear, direct, second person optional ("you") for how-to content.
- Practitioner-friendly; define jargon once, then use it consistently.
- No hype, no "best in class," no engagement bait.

## Structure

### Resources

1. Quick answer (40–60 words) — snippet target
2. Intro (80–120 words) — audience and context
3. 4–6 body sections with H2 headings
4. 2–3 examples in `examples[]`
5. 3–5 items in `commonMistakes[]`

### Guides

1. Intro — what the reader will learn
2. 6–8 progressive sections
3. Worked scenario with full numbers mid-article
4. "Next steps" section linking workflow tools
5. 3–5 FAQs where People Also Ask questions exist
6. YMYL disclaimer footer when applicable

## SEO

- Primary keyword in title, meta description, and first paragraph (natural phrasing).
- Do not create indexed numeric variation pages; link to canonical tool URLs.
- Do not duplicate the same paragraph structure across dozens of articles.

## Fact-checking

- Cross-check formulas against tool logic modules before publish.
- Cite standards in prose (WHO for BMI, ACI for concrete, etc.).
- Set `lastReviewed` on every publish or substantive edit.
