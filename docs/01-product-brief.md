# ConvertMyStuff Product Brief

## Purpose

ConvertMyStuff.com is a public, SEO-focused converter, calculator, generator, and utility website.

The primary product is not a login-first SaaS dashboard. The primary product is a set of fast, useful, indexable public tool pages that solve one task immediately without requiring account creation.

The site should eventually support optional premium features, but normal public tool use must remain available without login.

## Core Strategy

The site should start with practical long-tail utility tools rather than trying to compete directly with the largest incumbents in highly competitive PDF and media-conversion SERPs.

The first strategic wedge should include:

- Developer and data converters
- Text-cleanup utilities
- Practical unit and measurement converters
- Construction and material calculators
- Selective image utilities
- Real-estate calculators
- Marketing and SEO utilities

These categories are preferred because many tools are deterministic, relatively simple to build, and aligned with immediate task intent.

## Product Positioning

ConvertMyStuff should feel like a clean, fast, trustworthy utility site.

Users should be able to land on a tool page from Google, complete the task quickly, and leave satisfied. The page should also provide enough supporting information to build trust and rank organically, including examples, formulas, assumptions, FAQs, related tools, and source notes where relevant.

## Core Rules

1. Public tools should work without login.
2. Tool utility must appear above SEO content.
3. The codebase should scale through a reusable tool registry and shared page template.
4. The architecture should be ready for future accounts, payments, API access, batch processing, file storage, and admin features.
5. Do not overbuild the SaaS layer before the public tool platform is validated.

## Target Users

Primary users:

- Developers converting and formatting structured data
- Office workers cleaning text, CSVs, and files
- Homeowners and contractors estimating materials
- Real estate investors and agents using finance calculators
- Marketers using UTM, robots.txt, sitemap, and character tools
- General users needing quick file, image, unit, or text conversions

## Business Model

Initial monetization:

- Display ads on public tools
- Contextual affiliate links where appropriate
- Email capture only where it does not interfere with tool usage

Later monetization:

- Paid batch conversions
- Higher file limits
- API access
- No-ad subscription
- Saved history and projects
- Branded exports
- Downloadable reports
- Premium real-estate analysis tools

## Non-Goals for Version One

Do not build the following as core v1 requirements:

- Mandatory login
- Team workspaces
- Full SaaS dashboard
- Complex RBAC
- Heavy background jobs for media/OCR conversion
- Enterprise API platform
- Full CMS/admin panel
- Multi-tenant organization billing

These can be added later after traffic and usage patterns justify the complexity.
