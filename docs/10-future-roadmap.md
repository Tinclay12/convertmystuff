# Future Roadmap

## Guiding Principle

Build the public utility platform first, then add SaaS-style features only when they support proven user demand or revenue opportunities.

## Phase 1: Public Tool Platform

Goal:

Launch the core site architecture and first tools.

Includes:

- Homepage
- Category hubs
- Tool registry
- Public tool pages
- Dynamic routing
- Related tools
- Sitemap
- Robots.txt
- Basic schema
- Analytics placeholder
- Ad placeholders
- 5 to 10 initial tools

No required login.

## Phase 2: Content and Tool Expansion

Goal:

Expand traffic footprint and strengthen topical clusters.

Includes:

- More developer/data tools
- More text utilities
- More unit converters
- Construction calculators
- Image utilities
- Marketing tools
- Better examples and FAQs
- More internal linking
- Content quality review process

## Phase 3: Monetization Layer

Goal:

Start monetizing traffic without reducing utility.

Includes:

- Display ad integration
- Affiliate placements where appropriate
- Email capture experiments
- Upgrade prompt components
- Category-specific monetization testing

No login required for normal public use.

## Phase 4: Optional Accounts

Goal:

Add convenience features for repeat users.

Includes:

- Login
- Saved history
- Favorite tools
- Saved settings
- Saved templates
- Basic user dashboard

Accounts remain optional.

## Phase 5: Paid Features

Goal:

Monetize heavy usage and premium workflows.

Includes:

- Stripe integration
- No-ad plan
- Larger limits
- Batch conversions
- Premium exports
- Saved projects
- API keys
- Usage tracking

Login required only for paid/account features.

## Phase 6: Server-Side Processing and Jobs

Goal:

Support heavier tools that cannot reliably run in the browser.

Includes:

- File storage
- Queue/background job system
- Large image processing
- OCR
- PDF conversions
- Media conversions
- Batch processing
- Job status pages

Use this phase only when there is a clear traffic or revenue case.

## Phase 7: Admin CMS

Goal:

Make tool and SEO management easier without editing code.

Includes:

- Admin login
- Tool editor
- Category editor
- FAQ editor
- Example editor
- Source note editor
- Publishing workflow
- Noindex/draft controls
- Last reviewed tracking

## Phase 8: Advanced Programmatic Expansion

Goal:

Scale safely into more page families.

Includes:

- Additional format pairs
- Material-specific calculators
- Geo-specific calculators where sourced carefully
- Platform-specific tools
- More structured data
- Section-specific sitemaps

Do not create thin pages or indexed pages that are only numeric variations of the same conversion.

## Long-Term Vision

ConvertMyStuff can evolve into a broad utility platform with:

- Free public tools
- Strong organic search footprint
- Optional accounts
- Paid high-limit processing
- API access
- Niche premium calculators
- Admin-managed content
- Programmatic but quality-controlled page expansion

The public no-login utility layer should remain the foundation even after premium features are added.
